1.Apa perbedaan Master File dengan Transaction File?
jawab: Jika ditinjau dari segi perubahan isi dataya (data value), maka master file relatif lebih jarang berubah ketimbang transaction file. Contoh isi dari file MAHASISWA akan relatif lebih lama berubah jika dibanding isi dari file KRS (Kartu Rencana Studi).Jika ditinjau dari klasifikasi data, maka master file termasuk jenis (kelompok) data tetap (yang isinya relatif jarang berubah), sedangkan transaction file masuk ke jenis (kelompok) data tidak tetap (isinya relatif sering berubah, contoh KRS), dan masuk ke jenis kelompok data yang bertambah menurut waktu (contoh Transkrip Nilai).

Y = Jumlah siswa yang lolos seleksi berkas

Y = 400 x 21 / 20 = 420 Siswa yang lolos seleksi berkas

105/100 dari X = 630

X = 630 x 100 / 105

16. Berikan gambaran tentang sistem berkas dalam Penerapan dalam Bahasa Pemrograman! ​


SISTEM BERKAS

==================================

KONSEP DASAR SISTEM BERKAS

Sistem berkas merupakan mekanisme penyimpanan on-line serta untuk akses, baik data maupun program yang berada dalam sistem operasi.

------------------------------------------

Terdapat dua bagian penting dalam sistem berkas, yaitu:

- Kumpulan berkas, sebagai tempat penyimpanan data

- Struktur direktori, yang mengatur dan menyediakan informasi mengenai seluruh berkas dalam sistem.

------------------------------------------

Sistem Berkas

Sebuah sistem berkas sangat membantu para programmer untuk memungkinkan mereka mengakses berkas, tanpa memperhatikan detail dari karakteristik dan waktu penyimpanan. Sistem berkas ini juga yang mengatur direktori, device access dan buffer.

------------------------------------------

Tugas dari Sistem Berkas

- Memelihara direktori dari identifikasi berkas dan lokasi informasi.

- Menentukan jalan (pathway) bagi aliran data antara main memory dan alat penyimpanan sekunder.

- Mengkoordinasi komunikasi antara CPU dan alat penyimpanan sekunder, dan sebaliknya.

- Menyiapkan berkas penggunaan input atau output.

- Mengatur berkas bila penggunaan input atau output telah selesai.

------------------------------------------

BERKAS DAN AKSES

Sistem penyimpanan, pengelolaan dan penyimpanan data pada alat penyimpanan eksternal. Pada berkas dan akses penyimpanan data dilakukan secara fisik.

------------------------------------------

Berkas

- Kumpulan dari record-record yang saling berhubungan.

- Kumpulan dari statement-statement yang saling berhubungan

- Kumpulan informasi terkait yang diberi nama dan direkam pada penyimpanan sekunder

Pembahasan

Lampiran merupakan salah satu struktur surat dinas yang sifatnya optional (bisa dicantumkan atau tidak) dengan maksud menyampaikan segala sesuatu yang berkaitan dengan surat dinas tersebut.

Lampiran berasa dari kata "lampir" yang artinya menyertai segala sesuatu bersamaan dengan surat yang dimana yang berhubungan dengan surat tersebut. Lampiran juga didefinisikan sebagai lembar tambahan dari surat. Di dalam surat tersebut, harus dijelaskan diakhir surat seperti : "Mengenai hal ini, kami lampirkan jadwal study tour selama 11 hari"

✎ Contoh Lampiran Surat ✎

Lampiran surat dapat berupa seperti berikut :

Jadwal pelaksanaan (jika berupa acara seperti study tour, camping)Teks pendukungDokumen pendukung dari dokumen utamaDaftar riwayat hidup untuk surat lamaran pekerjaanGambar yang berkaitan dengan surat

30. Berikan gambaran tentang sistem berkas dalam penerapan dalam bahasa pemrograman


Jawaban:

SISTEM BERKAS

==================================

KONSEP DASAR SISTEM BERKAS

Sistem berkas merupakan mekanisme penyimpanan on-line serta untuk akses, baik data maupun program yang berada dalam sistem operasi.

------------------------------------------

Terdapat dua bagian penting dalam sistem berkas, yaitu:

- Kumpulan berkas, sebagai tempat penyimpanan data

- Struktur direktori, yang mengatur dan menyediakan informasi mengenai seluruh berkas dalam sistem.

------------------------------------------

Sistem Berkas

Sebuah sistem berkas sangat membantu para programmer untuk memungkinkan mereka mengakses berkas, tanpa memperhatikan detail dari karakteristik dan waktu penyimpanan. Sistem berkas ini juga yang mengatur direktori, device access dan buffer.

------------------------------------------

Tugas dari Sistem Berkas

- Memelihara direktori dari identifikasi berkas dan lokasi informasi.

- Menentukan jalan (pathway) bagi aliran data antara main memory dan alat penyimpanan sekunder.

- Mengkoordinasi komunikasi antara CPU dan alat penyimpanan sekunder, dan sebaliknya.

- Menyiapkan berkas penggunaan input atau output.

- Mengatur berkas bila penggunaan input atau output telah selesai.

------------------------------------------

BERKAS DAN AKSES

Sistem penyimpanan, pengelolaan dan penyimpanan data pada alat penyimpanan eksternal. Pada berkas dan akses penyimpanan data dilakukan secara fisik.

------------------------------------------

Berkas

- Kumpulan dari record-record yang saling berhubungan.

- Kumpulan dari statement-statement yang saling berhubungan

- Kumpulan informasi terkait yang diberi nama dan direkam pada penyimpanan sekunder
